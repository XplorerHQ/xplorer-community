# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Xplorer, please report it responsibly.

**Do not open a public issue for security vulnerabilities.**

Instead, please email: security@xplorer.dev

<!-- TODO: Set up security email or use GitHub's private vulnerability reporting -->

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes (optional)

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial Assessment**: Within 1 week
- **Resolution Target**: Depends on severity

## Scope

This security policy covers:
- The Xplorer CLI binary
- The Xplorer TUI application

Out of scope:
- Issues in dependencies (report to upstream)
- Issues in Kubernetes or Crossplane itself

## Supported Versions

Only the latest release is supported with security updates.

| Version | Supported |
|---------|-----------|
| Latest  | Yes       |
| Older   | No        |
